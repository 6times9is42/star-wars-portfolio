# Favicon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a lightsaber SVG favicon that switches from cyan to red when sith mode activates.

**Architecture:** Two static SVG files (one per theme) live in `images/`. The `<link rel="icon">` tag is given an id so JS can swap its `href` inside the existing `triggerThemeSwitch` timeout — at the same moment the `sith-mode` class is toggled on `<body>`.

**Tech Stack:** SVG, HTML, vanilla JS (no build step, no dependencies)

### Task 3: Swap favicon on sith mode toggle

**Files:**
- Modify: `script.js:771` (inside the 470ms timeout in `triggerThemeSwitch`)

- [ ] **Step 1: Add the favicon swap**

In `script.js`, the relevant block currently reads (lines 769–772):

```js
  /* Theme switches while still in wave 2 flash */
  setTimeout(() => {
    document.body.classList.toggle('sith-mode', toSith);
  }, 470);
```

Change it to:

```js
  /* Theme switches while still in wave 2 flash */
  setTimeout(() => {
    document.body.classList.toggle('sith-mode', toSith);
    document.getElementById('favicon').href = toSith
      ? 'images/favicon-sith.svg'
      : 'images/favicon-jedi.svg';
  }, 470);
```

- [ ] **Step 2: Verify the swap**

Open `index.html` in a browser. Trigger sith mode (the easter egg). Expected: browser tab icon switches from cyan to red at the same moment the page colours flip. Trigger it again to return to Jedi mode. Expected: tab icon returns to cyan.

- [ ] **Step 3: Commit and push**

```bash
git add script.js
git commit -m "feat: swap favicon on sith/jedi mode toggle"
git push origin main
```
